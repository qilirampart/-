# 帧析操作手册

## 1. 软件是做什么的

帧析是一款用于从视频、图片、分享链接和公众号文章中提取内容的桌面工具，主要解决以下几类需求：

- 从短视频中提取字幕、提示词、画面文字
- 从已经截好的图片中直接识别文字
- 通过抖音、快手、小红书、B站、微信公众号文章链接下载或解析内容
- 将视频中的音频分离出来，再转写成文本和字幕

软件目前分为 4 个工作区：

- `素材`：导入本地文件或通过链接下载内容
- `提取`：对图片或视频帧做 OCR 识别
- `语音转写`：把视频中的人声转成文本和 SRT 字幕
- `结果`：查看、复制、导出最终结果

建议使用顺序：

1. 先在 `素材` 页准备内容
2. 需要提取画面文字时进入 `提取`
3. 需要提取人声内容时进入 `语音转写`
4. 最后在 `结果` 页查看和导出

---

## 2. 启动前准备

### 2.1 Windows 便携版怎么用

如果你拿到的是打包好的压缩包：

1. 先完整解压，不要直接在压缩包里运行
2. 进入解压后的 `zhenxi` 文件夹
3. 双击 `zhenxi.exe` 启动

注意：

- 不建议把软件放在系统保护目录里，例如 `C:\Program Files`
- 建议放在普通目录，例如 `D:\软件\帧析`
- 第一次运行时，软件会自动创建 `runtime` 和 `output` 目录

### 2.2 运行环境说明

软件默认更推荐使用云端能力，而不是本地大模型。

当前建议：

- 画面文字识别：优先使用云端 OCR API
- 语音转写：使用腾讯云语音识别
- 本地 OCR：仅作为兼容入口保留，不建议优先使用

原因很简单：

- 云端识别通常更稳定
- 不需要手动维护本地模型
- 打包版体积更可控
- 遇到显卡、模型下载、系统兼容问题时更省事

## 3. 界面总览

### 3.1 顶部状态区

主窗口右上角通常会显示：

- `使用说明`
- `API 配置`
- 当前状态提示，例如“提取完成”

你可以把它理解成全局控制区：

- `使用说明`：打开本手册
- `API 配置`：配置 OCR API

### 3.2 四个工作区

#### 素材

负责“拿到内容”。

可以做的事：

- 导入本地视频
- 导入本地图片
- 粘贴分享文本后下载素材
- 清空当前素材

#### 提取

负责“从画面里拿文字”。

可以做的事：

- 播放视频
- 逐帧查看
- 自动选帧
- 手动补帧
- 手动框选 OCR 区域
- 开始静态提取

#### 语音转写

负责“从声音里拿文字”。

可以做的事：

- 使用当前视频
- 选择视频或音频
- 分离音频
- 调用腾讯云接口转写
- 导出 TXT、SRT、JSON

#### 结果

负责“整理和导出结果”。

可以查看：

- 完整结果
- 按帧结果
- 状态日志

## 4. 快速上手

如果你第一次使用，建议直接照这个流程：

### 4.1 提取视频画面中的文字

1. 打开软件
2. 在 `素材` 页点击 `导入视频`，或者直接粘贴分享文本后点击 `粘贴下载`
3. 导入完成后点击右侧 `进入提取`
4. 在 `提取` 页播放视频，确认内容正确
5. 选择选帧方式
6. 需要时启用 `手动框选 OCR 区域`
7. 点击 `开始静态提取`
8. 提取完成后去 `结果` 页查看内容
9. 需要导出时点击 `导出 TXT` 或 `导出 JSON`

### 4.2 把视频中的人声转成字幕

1. 在 `素材` 页先导入视频
2. 切换到 `语音转写`
3. 点击 `使用当前视频`
4. 如果还没配置腾讯云，先点击 `音频 API 配置`
5. 点击 `分离音频`
6. 点击 `开始转写`
7. 在下方结果页查看：
   - 完整文本
   - 字幕 SRT
   - 处理日志
8. 需要时导出 TXT、SRT 或 JSON

### 4.3 直接识别一张图片里的文字

1. 在 `素材` 页点击 `导入图片`
2. 切换到 `提取`
3. 如有需要，手动框选识别区域
4. 点击 `开始静态提取`
5. 在 `结果` 页查看最终文本

## 5. 素材页详细说明

素材页的作用是统一处理“输入来源”。

### 5.1 导入视频

适用场景：

- 你本地已经有视频文件
- 你想做视频逐帧提取
- 你想从视频中分离音频

支持常见格式：

- `.mp4`
- `.mov`
- `.avi`
- `.mkv`
- `.flv`
- `.wmv`
- `.m4v`

### 5.2 导入图片

适用场景：

- 你已经提前截好了图
- 你只想识别单张图片里的文字
- 不需要再从视频里选帧

支持常见格式：

- `.png`
- `.jpg`
- `.jpeg`
- `.webp`
- `.bmp`

### 5.3 粘贴下载

适用场景：

- 你手里只有平台分享文本
- 不想先手动下载再导入

当前已支持或已集成解析能力的平台：

- 抖音视频
- 快手视频
- 小红书视频
- B站视频
- 微信公众号文章

说明：

- 微信公众号文章当前主要提取正文文本和图片，不以视频完整下载为主要目标
- 不同平台会受到风控、地区、分享口令时效、登录态等影响，偶尔失败属于正常现象

### 5.4 粘贴下载的正确方式

推荐做法：

1. 在手机或网页里复制完整分享文本
2. 回到软件
3. 直接点击 `粘贴下载`

软件下载时会优先读取剪贴板内容；如果剪贴板没有可用文本，也会读取当前输入框里的内容。

### 5.5 取消下载

如果下载过程中发现链接错误、资源过大或网络异常，可以点击 `取消下载`。

取消后通常会：

- 停止当前下载线程
- 删除未完成的临时文件
- 保留之前已经成功导入的素材

## 6. 提取页详细说明

提取页的核心目标，是从画面中提取文字。

### 6.1 什么时候用提取页

以下情况都应该进入 `提取` 页：

- 想识别短视频里的字幕
- 想提取画面中的提示词
- 想从多帧中综合得到一份完整结果
- 想排除水印、头像、贴纸等干扰区域

### 6.2 视频预览区

视频预览区是整个提取流程的核心区域。

你可以在这里：

- 播放 / 暂停视频
- 切换上一帧 / 下一帧
- 拖动时间轴
- 观察画面细节
- 手动框选 OCR 区域

如果视频中的文字比较小，建议：

- 先暂停到合适位置
- 再逐帧查看
- 必要时缩小识别区域，只框文字区域

### 6.3 选帧模式

提取页提供三种选帧模式：

#### 自动选帧 + 手动补充

适合大多数情况，推荐优先使用。

特点：

- 软件先自动生成候选帧
- 你再手动补几帧或删几帧
- 效率和可控性比较平衡

#### 仅自动选帧

适合画面变化比较规律、字幕比较稳定的视频。

特点：

- 操作最省事
- 但如果自动选帧漏掉关键帧，需要重新处理

#### 仅手动选帧

适合以下情况：

- 你只想识别一两帧
- 视频里文字出现时间很短
- 自动选帧容易选错

### 6.4 候选帧管理

你可以通过以下按钮管理候选帧：

- `生成候选帧`
- `加入当前帧`
- `删除选中帧`

建议做法：

1. 先生成候选帧
2. 检查候选列表是否覆盖关键画面
3. 如有遗漏，暂停到目标位置后点击 `加入当前帧`
4. 删除明显无关或重复的帧

### 6.5 手动框选 OCR 区域

这个功能很重要，尤其在画面复杂时。

适合场景：

- 只识别字幕区域
- 避开右上角水印
- 排除头像、贴纸、弹幕、边框
- 画面中只有局部区域才有有效文字

使用建议：

- 尽量框得紧一些
- 不要把无关背景一起框进去
- 如果视频中字幕位置有明显变化，要重新检查框选是否还对得上

### 6.6 OCR 引擎选择

当前更推荐使用：

- `视觉 API（云端）`

源码版在非打包环境下，可能还能看到本地 OCR 选项，但默认不建议优先使用。

原因：

- 云端 OCR 效果通常更稳定
- 本地模型需要额外下载和维护
- 本地 OCR 对运行环境更敏感

### 6.7 提取完成后去哪看

提取完成后，结果会出现在：

- `结果` 页

你也可以点击提取页里的 `查看结果`，直接切过去。

## 7. 语音转写页详细说明

语音转写页用于把视频中的人声转成文字和字幕。

### 7.1 这个页面能做什么

可以完成以下流程：

1. 从视频中分离音频
2. 自动按长度切分较大的音频
3. 将音频提交到腾讯云识别
4. 获取完整文本
5. 生成 SRT 字幕
6. 导出 TXT / SRT / JSON

### 7.2 支持两种输入方式

#### 使用当前视频

最推荐。

适用场景：

- 你已经在素材页导入了视频
- 希望少一次重复选择文件

#### 手动选择视频或音频

适合以下情况：

- 你要单独处理别的视频
- 你手头已经有分离好的音频文件

### 7.3 分离音频

点击 `分离音频` 后，软件会：

1. 从视频中提取音轨
2. 保存到本地音频目录
3. 如果音频过大，会自动分段

这一步完成后，你会在页面上看到：

- 音频输出路径
- 时长
- 文件大小
- 分段数量

### 7.4 开始转写

点击 `开始转写` 后，软件会：

1. 读取腾讯云配置
2. 把音频或分段音频提交到接口
3. 轮询结果
4. 合并所有段落
5. 生成完整文本和 SRT 字幕

### 7.5 转写结果页签

语音转写页下方通常有三个页签：

#### 完整文本

适合：

- 复制正文
- 进一步整理文案
- 导出 TXT

#### 字幕 SRT

适合：

- 导入剪辑软件
- 做字幕校对
- 导出标准字幕文件

#### 处理日志

适合排查问题，例如：

- 是否已经分离音频
- 分成了几段
- 哪一步失败
- 最终保存到了哪里

### 7.6 中断转写

如果转写时间较长，或者你发现选错素材，可以点击 `中断转写`。

注意：

- 中断后当前任务会停止
- 已经得到的结果不会自动变成最终可用结果
- 建议重新确认素材后再发起一次完整转写

## 8. OCR API 配置

OCR API 配置用于画面文字识别。

入口：

- 主窗口右上角 `API 配置`

### 8.1 支持多通道

当前支持最多 5 个 API 通道。

你可以：

- 配置多个服务
- 手动指定首选通道
- 在首选失败时自动降级到后续可用通道
- 单独测试每个通道是否可连接

### 8.2 建议配置方式

建议至少保留 2 个可用通道：

- 一个主通道
- 一个备用通道

这样当首选服务超时、限流或失效时，软件还能自动尝试下一个。

### 8.3 配置时要填什么

通常需要：

- 通道名称
- Base URL
- API Key
- 模型名称
- 超时时间
- OCR 提示词

### 8.4 测试连接是什么意思

`测试当前通道` 只负责验证：

- 地址是否可访问
- Key 是否大致可用
- 模型接口是否有响应

它不代表实际 OCR 一定成功。

也就是说：

- `测试成功` 说明通道基本通了
- 但正式提取时仍可能因模型能力、接口兼容性、返回格式等问题失败

### 8.5 常见配置建议

- Base URL 通常要带上 `/v1`
- 模型名必须和服务端支持的名字一致
- 如果接口是 OpenAI 兼容格式，优先使用兼容更好的模型

## 9. 音频 API 配置

音频 API 配置用于语音转写。

入口：

- `语音转写` 页里的 `音频 API 配置`

### 9.1 当前使用什么服务

目前语音转写使用的是腾讯云录音文件识别接口。

### 9.2 需要填写的内容

通常包括：

- 是否启用腾讯云语音转写
- SecretId
- SecretKey
- Region
- 引擎模型
- 返回格式
- 声道数

### 9.3 推荐配置

一般情况下建议：

- Region：`ap-shanghai`
- 引擎：`16k_zh`
- 返回格式：`3`
- 声道数：`1`

这个组合更适合普通中文视频转写和字幕输出场景。

### 9.4 什么情况下会失败

常见原因：

- 腾讯云服务未开通
- SecretId / SecretKey 填错
- 账号无资源包且未开后付费
- 音频识别接口暂时异常
- 网络环境无法访问腾讯云

## 10. 结果页详细说明

结果页主要用于统一查看提取结果。

### 10.1 完整结果

这是整理后的最终文本。

适合：

- 直接复制
- 作为最终输出
- 导出 TXT

### 10.2 按帧结果

这是按候选帧拆分的 OCR 结果。

适合：

- 回看某一帧识别了什么
- 排查哪些帧识别对了，哪些帧识别错了
- 判断是否需要补帧或删帧

### 10.3 状态日志

用于记录整个处理过程。

建议在以下情况查看：

- 为什么下载失败
- 为什么提取失败
- 为什么结果为空
- 输出文件到底保存在哪里

### 10.4 导出功能

结果页常用导出方式：

- `导出 TXT`
- `导出 JSON`

语音转写页还额外支持：

- `导出 SRT`

---

## 11. 输出文件保存位置

软件运行后会在程序目录下生成 `output` 文件夹。

常见目录如下：

- `output/downloads`
  - 下载得到的视频
- `output/screenshots`
  - OCR 提取过程中的截图或中间文件
- `output/audio`
  - 从视频中分离出来的音频
- `output/transcripts`
  - 默认导出的转写结果
- `output/logs`
  - 运行日志

你可以把这些目录理解为“软件自动工作区”。

如果你想找刚下载的视频、刚分离的音频、刚转写的字幕，优先去 `output` 里看。

## 12. 推荐工作流

### 12.1 短视频提示词提取

适合抖音、快手、小红书、B站视频。

推荐流程：

1. 粘贴分享文本并下载
2. 进入提取页
3. 自动选帧
4. 手动删掉无关帧、补上关键帧
5. 必要时框选 OCR 区域
6. 开始静态提取
7. 去结果页复制最终文本

### 12.2 已经截图好的图片提取

推荐流程：

1. 导入图片
2. 进入提取页
3. 如有需要，框选识别区域
4. 开始静态提取
5. 查看并导出结果

### 12.3 视频口播转文字

推荐流程：

1. 导入视频
2. 进入语音转写
3. 使用当前视频
4. 分离音频
5. 开始转写
6. 导出 TXT 或 SRT

### 12.4 公众号内容整理

推荐流程：

1. 在素材页粘贴微信公众号文章链接
2. 软件下载并解析文章内容
3. 查看输出目录中的正文和图片
4. 按需人工整理

## 13. 常见问题

### 13.1 为什么链接有时能下，有时不能下

常见原因：

- 分享口令过期
- 平台风控
- 当前网络环境异常
- 目标平台临时限制
- 内容类型不是当前下载器处理的那一种

建议：

- 尽量复制完整分享文本
- 失败后重新复制一次
- 如果是代理环境，尽量保持网络稳定

### 13.2 为什么测试 API 成功，但正式提取失败

因为“测试连接”和“正式提取”不是一回事。

测试连接只说明：

- 服务端可访问
- Key 大体有效

正式提取还会受到这些因素影响：

- 模型是否真支持图像 OCR
- 返回格式是否兼容
- 当前模型权限是否足够
- 请求是否超时

### 13.3 为什么结果页没有内容

优先检查：

1. 是否真的选中了候选帧
2. 框选区域是否框到了文字
3. 文字本身是否清晰
4. 当前 API 通道是否可用
5. 提取日志里是否已经报错

### 13.4 为什么本地 OCR 不推荐优先使用

因为它通常更麻烦：

- 需要模型文件
- 对机器环境更敏感
- 启动更重
- 实际效果未必比云端更好

所以当前版本更推荐云端 OCR。

### 13.5 为什么语音转写有时比较慢

因为它不是本地实时识别，而是：

1. 先分离音频
2. 再上传到腾讯云
3. 再轮询等待结果

素材越长，等待时间越久。

---

## 14. 使用建议

如果你的目标是“稳定拿到结果”，推荐遵循下面这套顺序：

1. 优先使用完整分享文本，而不是只贴短链接
2. 优先使用云端 OCR
3. 提取前先把候选帧整理干净
4. 画面复杂时一定要框选 OCR 区域
5. 语音转写前先确认腾讯云配置可用
6. 提取失败时先看状态日志，不要盲目重复点很多次

## 15. 一句话总结

可以把帧析理解成一套“先拿素材，再分成画面文字提取和语音转写两条路，最后统一导出结果”的工具。

最常见的用法只有三步：

1. 导入或下载素材
2. 选“提取”或“语音转写”
3. 到“结果”里复制或导出

如果你是第一次使用，最推荐先从一条短视频开始测试完整流程，跑通一次后再批量处理。
